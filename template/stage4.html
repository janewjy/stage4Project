1.3: Adding CSS Style and HTML Structure

Avoiding repetition

Avoiding repetition is the essential of programming. Cascading style sheet (CSS) let the coder avoid repetition.
Avoiding Errors: Without avoiding repetition, the code that do the same thing will often repeat over and over, like copying and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they do not diligently make that same change everywhere the repeated code appears, problems will arise.
Consistency: Consistency is a key to a well-organized and elegant web page. The same style can be assigned to each div and specified in only one place! It is also easy to make changes on that style.

Understand CSS and thinking about inheritance

CSS allows the descendant elements inherit property values from ancestor elements. Properties that can be inherited are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing. Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index. This wiki-page details inheritance of CSS.

Set the attributes in CSS for different tags or classes

The style of each element can be set by assigning attributes to tags or classes.

Web design process

  • Look for natural boxes
  • Look for repeated styles and semantic elements
  • Write the HTML
  • Apply style (from biggest to smallest)
  • Fix details

2.3: Input → Function → Output

What is a function?

A function is a series of operations on input and then produce output. There is a python function example below:

def restOfString(s):
  return s[1:]

This function returns the rest of a string other than its first letter.

What is the difference between making and using a function?

Making a function is defining the action and parameters inside the function.
Using a function is, after making the function, calling this function to do some certain task

How do functions help programmers avoid repetition?

Functions only need to be defined once and allow use to operate the same action with different input.

What happens if a function does not have a return statement?

return returns the output after calling the function. Without return, although the function modified the input, but no output from the function and nothing would happen.

2.5: Structure Data & How to Solve Problems

List

String and list are two kinds of structure data. Strings only contain a sequence of characters while list can be sequences of anything. list can be inside another list. List support mutation which means the value of the list can be changed after created. If both names refer to the same list, any changes affect one value name refers to also affect the values the other one refers to.

Solving complex problem

Don't panic! Identify the inputs and how they are represented. Then identify the outputs. Understand the relationship between the input and output with some examples. Test your thoughts with a simple mechanical solution. When it works, develop incrementally and test the results.

3.1-3.4: Add videos to your page

Object Oriented Programming

Object-oriented programming (OOP) allows the user to build an object or instance in an intuitive way with some certain variables and attributes from a class. OOP let the uses coding in a way resemble human thinking process. In general, when we try to learn from and solve problems in the world, the object or problem with similar attributes are always grouped together. In the coding process, OOP is a valuable tool because the objects sharing the similar attributes can be created from the same class. If more constraints are needed on a class of objects, then the child class can inherit attributes from its parent class and also add its own attributes and functions in its own class. With such features, OOP can avoid coding repetition, same time and energy for the programmer.

Important terms

  • Object: structures that contain predefined variables and functions.
  • Class: a template for creating objects which contains constructor for objects, available procedures and variables.
  • Instance: also called object which is always referred to a specific thing. For example, skin care product is a class (contains the functions and attributes sharing among skin care products), facial cream is a child class (such class contains the same attributes form skin care product adding along with some special attributes that only for face), Jingya's home made daytime facial cream is a instance which contains all the details of functions, ingredients and parameters of such product.
  • Method: is a function that can be operated on the object.
  • Module: is an element that contains one aspect of functions, classes and variables that can be implemented for some certain purpose.
  • Library: a collection of modules.

Abstract thinking

Instead of considering only one specific object. OOP and CSS both take one step back. Observe and process object from a generalization view. For example, classes defined the attributes and operations that are shared among specific objects. With the features such as inheritances and method overriding for classes, OOP can efficiently avoid code repetition.

Server

To request a doc, the browser will send a request line (usually follow the HTTP) to the server: GET/url HTTP/version_info. The request is followed by a number of headers, which contains host and user-agent information.

After the server revieve a request, the server will response with a status line: HTTP/version_info status_code reason_phrase. The content of static is pre-written files that the server just returns, such as images. Dynamic content is made on the fly which is very popular nowadays.
